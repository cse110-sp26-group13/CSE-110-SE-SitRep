## Sprint Summary

Sprint 1 focused on domain research, feature selection, and initial architecture decisions. The team went from project kickoff to a defined MVP scope, a bare-bones project skeleton, and the start of feature implementation by the end of the sprint.

**Delivered:**
- Completed domain research (ClickUp, Linear, Range, Trello, GeekBot, etc.)
- Voted on and finalized MVP feature set: Daily Standup, Onboarding, Calendar, GitHub Issue Tracker
- Agreed on project skeleton (`shazi/barebones` branch) reviewed by all members
- Divided team into sub-groups per feature
- Started implementation: Daily Standup (Yang), Onboarding/backend (Shazi), GitHub Issue Tracker research (Andrew, Ophir, Jeong)

---

## What Went Well ✅

- **Strong research phase** — The team explored a wide range of reference products and identified genuinely useful features (burnout tracker, activity auto-sync, AI digest) that informed the MVP.
- **Clear MVP scoping** — The team converged on 4 concrete MVP features with assigned owners, which gave everyone a clear direction heading into implementation.
- **Project structure agreement** — All members reviewed and aligned on the bare-bones skeleton before coding started, preventing early divergence.
- **Sub-team formation** — Splitting into feature-based sub-groups (3-person for Issue Tracker, 2-person for Calendar, etc.) allowed parallel progress.
- **Good attendance** — Most meetings had near-full attendance despite midterm season.

---

## What Didn't Go Well ❌

- **Delayed feature decisions** — Feature selection was pushed from 5/12 to 5/14, compressing the time available to start implementation.
- **Incomplete research notes** — Some research items from 5/7 (GeekBot, When2Meet, From Notion/When2Meet sections) were left blank or underdeveloped.
- **AI integration left unresolved** — AI usage was discussed in 5/5 and 5/7 but remained undefined by sprint end with no concrete plan.
- **Standup meeting uploads pending** — As of 5/12, standup notes and research had not yet been uploaded to the GitHub repo, falling behind the documentation process requirement.
- **Branch workflow not yet established** — The rule "make a git issue before working" was only communicated at the 5/16 meeting, meaning earlier work may not have followed this process.
