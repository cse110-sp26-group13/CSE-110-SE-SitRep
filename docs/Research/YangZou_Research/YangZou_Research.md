## Slack

### User Status (custom status + time duration)

![image.png](Yang%20Zou/image.png)

- I like this feature because it allows users to quickly show what they are doing, like “in a meeting” or “commuting”.
- It is very clear and easy to understand, so team members know each other's situation without asking.
- The time duration is also useful because it tells how long the person will be busy.
- This helps reduce unnecessary messages and improves team communication.
- We can use this in our project to track team member status and activity.
- For example, users can update their current work, availability, or blockers.
- This is very important for agile teams because everyone needs to know team progress and availability.

### Channels

- Channels are very useful because they separate conversations by topic.
- This makes communication more clear and easy to manage.
- We can use channels in our app to separate different types of team updates or tasks.

### Notifications Control

- I like that Slack allows users to control notifications.
- Users can mute channels or customize alerts.
- This helps reduce distraction and improves focus.
- We can use this feature so users can control what updates they want to see.

## Discord

### Voice Channels

- I like this feature because users can join a voice channel anytime without calling or scheduling.
- It feels very natural, like just walking into a room and start talking.
- This makes communication faster and more flexible for teams.
- It is very useful when team members need quick discussion or help.
- We can use this in our project to allow team members to quickly communicate when needed.

### Screen Sharing

- I like this feature because users can share their screen in real time during a voice call.
- It is very helpful when explaining ideas, showing code, or solving problems together.
- Instead of typing a lot of messages, people can directly show what they are working on.
- This makes communication faster and more clear.
- We can use this in our project to support collaboration between team members.

## Trello

### Inbox

- I like this feature because it allows users to collect tasks from different platforms into one place.
- It helps organize all incoming tasks and ideas, so nothing gets lost.
- This is very useful when users receive tasks from email, messages, or other apps.
- We can use this in our project to collect team updates or tasks in one place.

### Add Cards

- I like this feature because it allows users to quickly add tasks.
- It is simple and fast, so users can write down ideas anytime.
- This helps users not forget important tasks.
- We can use this in our project to let users quickly create tasks or updates.

### Planner (calendar view + integration)

- I like this feature because it shows tasks in a calendar view.
- It helps users understand deadlines and schedule clearly.
- This makes it easier to plan work and manage time.
- I also like that it can connect with Google Calendar and Outlook.
- It allows users to sync their tasks and meetings automatically.
- This makes the system more convenient and efficient.
- We can use this in our project to show team tasks over time.

### Board View (My Trello Board)

- I like this feature because it shows all tasks in one board with different sections.
- For example, tasks can be grouped into “Today”, “This Week”, and “Later”.
- This makes it very easy to understand what needs to be done and when.
- It gives a clear overview of all tasks at a glance.
- The layout is simple and visual, so users can quickly find important tasks.
- It also helps users stay organized and focused on priorities.
- We can use this in our project to display team tasks and progress in one place.
- This feature provides a clear structure for task management and improves team visibility.
